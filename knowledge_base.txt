БАЗА ЗНАНИЙ TELEGRAM-БОТА ПО ВКР / БКП

Тема выпускной квалификационной работы:
«Разработка системы анализа русскоязычных текстов с выделением тематических групп».

Название разработанного приложения:
VSTU READS.

Тип системы:
Настольное программное приложение с графическим пользовательским интерфейсом для анализа русскоязычных текстов, выделения ключевых слов, определения тематической группы, сохранения результатов анализа, просмотра истории, построения аналитических диаграмм и экспорта отчетов в PDF.

Назначение системы:
Система VSTU READS предназначена для автоматизированного анализа русскоязычных текстов. Она позволяет пользователю вводить текст вручную или загружать его из файла, выполнять предварительную обработку, выделять ключевые слова, определять тематическую группу, сохранять результат анализа в локальной базе данных SQLite, просматривать историю выполненных анализов, анализировать распределение текстов по темам и экспортировать результаты в PDF.

Главная цель ВКР:
Целью выпускной квалификационной работы является разработка программной системы для анализа русскоязычных текстов и автоматического выделения тематических групп на основе методов предварительной обработки и интеллектуального анализа текстовых данных.

Актуальность темы:
Актуальность темы связана с постоянным ростом объема текстовой информации. Значительная часть информации представлена в виде неструктурированных русскоязычных текстов: учебных материалов, научных статей, документов, новостей и других источников. Ручной анализ таких данных требует много времени, зависит от субъективной оценки пользователя и не всегда позволяет быстро выделить ключевые признаки текста или определить его тематическую направленность. Поэтому необходимы программные средства, которые автоматизируют анализ русскоязычных текстов, выделяют ключевые слова, определяют тематические группы, сохраняют результаты и представляют их в удобной форме.

Практическая значимость:
Практическая значимость работы заключается в создании настольного приложения, которое помогает пользователю быстрее анализировать русскоязычные тексты, определять их тематическую направленность, выделять ключевые слова, сохранять результаты, просматривать историю и формировать отчеты. Система может использоваться в учебной, исследовательской, информационно-аналитической и деловой деятельности.

Объект исследования:
Объектом исследования являются русскоязычные текстовые данные, подлежащие тематическому анализу и группировке.

Предмет исследования:
Предметом исследования являются методы, алгоритмы и программные средства анализа русскоязычных текстов, обеспечивающие выделение тематических групп.

Задачи ВКР:
1. Провести анализ предметной области и существующих программных решений для обработки и тематической группировки текстов.
2. Выявить недостатки существующих аналогов и сформулировать требования к разрабатываемой системе.
3. Выбрать и обосновать методы предварительной обработки русскоязычных текстов и способы выделения тематических групп.
4. Спроектировать архитектуру программной системы, пользовательский интерфейс и структуру данных.
5. Реализовать настольное приложение для анализа русскоязычных текстов и выделения тематических групп.
6. Провести тестирование разработанной системы и оценить эффективность ее применения.

В работе были рассмотрены следующие аналоги:
1. IBM SPSS Modeler Text Analytics.
2. KNIME Analytics Platform с расширением Text Processing.
3. Orange Data Mining с модулем Text Mining.
4. Altair RapidMiner / AI Studio.

Вывод по анализу аналогов:
Существующие решения обладают широкими возможностями для обработки текстов, извлечения признаков и тематического анализа. Однако они часто являются коммерческими, сложными в освоении, требуют дополнительной настройки или не ориентированы специально на задачу анализа русскоязычных текстов в виде простого настольного приложения. Поэтому разработка собственной системы VSTU READS является обоснованной.

4. Нормализацию текстовой информации.
5. Выделение значимых признаков и содержательных характеристик текста.
6. Автоматическое формирование тематических групп.
7. Отображение результатов тематической группировки в понятной форме.
8. Просмотр исходных текстов и результатов обработки.
9. Сохранение результатов анализа.
10. Удобное взаимодействие пользователя с системой через графический интерфейс.
11. Просмотр истории сохраненных анализов.
12. Построение аналитических диаграмм.
13. Экспорт результатов в PDF.
14. Настройку темы оформления и языка интерфейса.

Входные данные:
Входными данными являются русскоязычные текстовые материалы, вводимые пользователем вручную или загружаемые из файла.

Выходные данные:
Выходными данными являются:
1. Тематическая группа текста.
2. Ключевые слова.
3. Краткое описание результата анализа.
4. Сохраненный результат анализа.
5. История анализов.
6. Диаграммы распределения текстов по тематическим группам.

7. Система должна быть совместима с персональным компьютером пользователя.

Особенности русскоязычных текстов:
Русскоязычные тексты сложны для автоматической обработки из-за богатой морфологии, большого количества словоформ, синонимии, омонимии, разнообразных синтаксических конструкций и контекстной неоднозначности. Поэтому в системе используются этапы очистки, токенизации, нормализации, удаления стоп-слов и выделения информативных признаков.

Методы обработки текста:
В системе применяются следующие этапы обработки:
1. Очистка текста от лишних символов, пунктуации, чисел и шумовых элементов.
2. Приведение текста к единому виду.
3. Токенизация — разбиение текста на отдельные элементы.
4. Нормализация слов для уменьшения влияния разных грамматических форм.
5. Удаление стоп-слов.
6. Выделение значимых признаков текста.
7. Определение тематической близости.
8. Формирование тематической группы.
9. Выделение ключевых слов.
10. Отображение и сохранение результата.

Алгоритм функционирования системы:
1. Пользователь вводит или загружает русскоязычный текст.
2. Система проверяет корректность входных данных.
3. Выполняется предварительная очистка текста.
4. Текст приводится к унифицированному виду.
5. Выделяются информативные текстовые единицы.
6. Исключаются малозначимые элементы и стоп-слова.
7. Формируется признаковое представление текста.
8. Определяется тематическая близость текста.
9. Формируется тематическая группа.
10. Пользователю отображаются результаты анализа.
11. Результат может быть сохранен в базе данных.
12. Пользователь может просмотреть историю, аналитику или экспортировать результат в PDF.

Архитектура системы:
Архитектура системы построена на модульном подходе. Модульная архитектура позволяет разделить приложение на логически завершенные части, упростить разработку, тестирование, сопровождение и дальнейшее расширение системы.

Основные модули системы:
1. Модуль пользовательского интерфейса.
2. Модуль авторизации.
3. Модуль ввода и загрузки текстов.
4. Модуль предварительной обработки текста.
5. Модуль анализа и выделения признаков.
6. Модуль тематической группировки текстов.
7. Модуль отображения результатов.
8. Модуль визуализации.
9. Модуль хранения данных.
10. Модуль экспорта.
11. Модуль настроек.

Модуль пользовательского интерфейса:
Обеспечивает взаимодействие пользователя с программой. Через него пользователь вводит текст, загружает файл, запускает анализ, просматривает результаты, сохраняет данные, открывает историю, аналитику, профиль, настройки и страницу «О программе».

Модуль авторизации:
Обеспечивает вход пользователя в приложение. В прототипе используется упрощенная модель авторизации: для входа достаточно ввести непустые значения email и пароля.

Модуль ввода и загрузки текстов:
Позволяет пользователю вводить текст вручную или загружать текстовые данные из файла. После получения текста модуль передает данные на дальнейшую обработку.

Модуль предварительной обработки:
Выполняет очистку текста, удаление лишних символов, приведение текста к единому виду и подготовку данных к анализу.

Модуль анализа и выделения признаков:
Формирует содержательное представление текста, выделяет значимые элементы и ключевые слова, которые используются для определения тематической группы.

Модуль тематической группировки:
Является ключевым модулем системы. Он определяет тематическую группу текста на основе содержательных признаков и смысловой близости.

Модуль отображения результатов:
Выводит пользователю тематическую группу, ключевые слова, краткое описание анализа и другие результаты обработки.

Модуль визуализации:
Строит диаграммы распределения сохраненных анализов по тематическим группам. В системе используются столбчатая и круговая диаграммы.

Модуль хранения данных:
Сохраняет исходные тексты, результаты анализа, тематические группы и ключевые слова в базе данных SQLite.

Модуль экспорта:
Позволяет экспортировать текущий результат анализа или аналитический отчет в PDF.

Модуль настроек:
Позволяет изменять тему оформления, язык интерфейса, очищать историю анализов и проверять состояние базы данных.

База данных:
В системе используется SQLite — легковесная встроенная база данных, подходящая для настольного приложения. SQLite не требует отдельного сервера и хранит данные локально на компьютере пользователя.

Основные таблицы базы данных:
1. texts — таблица исходных текстов.
2. topic_groups — таблица тематических групп.
3. analysis_results — таблица результатов анализа.

Поля таблицы texts:
1. text_id — уникальный идентификатор текста.
2. text_content — содержимое исходного текста.
3. source_type — тип источника текста.
4. created_at — дата и время добавления текста.

Поля таблицы topic_groups:
1. group_id — уникальный идентификатор тематической группы.
2. group_name — наименование тематической группы.
3. group_description — краткое описание тематической группы.

Поля таблицы analysis_results:
1. result_id — уникальный идентификатор результата анализа.
2. text_id — связь с исходным текстом.
3. group_id — связь с тематической группой.
4. keywords — ключевые слова, выделенные при анализе.
5. analysis_date — дата и время выполнения анализа.

Связи в базе данных:
Одному тексту может соответствовать одна или несколько записей о результатах анализа. Одной тематической группе могут соответствовать результаты анализа нескольких текстов. Таблица analysis_results связывает исходные тексты и тематические группы.

Используемые технологии в приложении VSTU READS:
1. Python — основной язык программирования.
2. PySide6 — библиотека для создания графического пользовательского интерфейса.
3. SQLite — локальная база данных.
4. Matplotlib — библиотека для построения диаграмм.
5. Visual Studio Code — среда разработки.
6. PDF-экспорт — функция формирования отчетов.
7. Библиотеки обработки текстовых данных Python.

Почему выбран Python:
Python выбран из-за простоты синтаксиса, высокой скорости разработки, большого количества библиотек для обработки текстов, удобной интеграции с GUI и базой данных, а также широкой применимости в задачах анализа данных и NLP.

Почему выбран PySide6:
PySide6 выбран для создания современного графического интерфейса настольного приложения. Он позволяет создавать окна, кнопки, текстовые поля, таблицы, панели навигации и другие элементы интерфейса.

Почему выбран SQLite:
SQLite выбран потому, что это легковесная встроенная СУБД, которая не требует отдельного сервера и подходит для настольного приложения. Она позволяет хранить исходные тексты, результаты анализа и тематические группы локально.

Почему выбран Matplotlib:
Matplotlib используется для построения аналитических диаграмм, которые помогают визуально показать распределение сохраненных текстов по тематическим группам.

Интерфейс приложения построен как настольное приложение с боковой навигационной панелью и несколькими функциональными страницами. Основные разделы: Home, History, Analytics, Profile, Settings и About.

Окно авторизации:

Главная страница Home:
Основное рабочее пространство. Пользователь вводит текст вручную или загружает файл, запускает анализ, получает тематическую группу, ключевые слова и краткий результат. Также доступны функции сохранения результата, очистки поля и экспорта в PDF.

Страница History:
Предназначена для просмотра сохраненных результатов анализа. В таблице отображаются фрагмент текста, тематическая группа, ключевые слова и дата анализа. Есть элементы Search, Refresh, Open и Delete.

Страница Analytics:
Показывает диаграммы распределения сохраненных анализов по тематическим группам. Используются столбчатая и круговая диаграммы.

Страница Profile:
Содержит информацию о текущем пользователе: email, имя пользователя, выбранная тема, язык интерфейса, количество сохраненных анализов и время последнего входа.

Страница Settings:
Позволяет менять тему оформления, язык интерфейса, очищать историю анализов, сбрасывать настройки и проверять состояние базы данных.

Страница About:
Содержит название программы, назначение приложения, тему ВКР, сведения о разработчике и версию программы.

TELEGRAM-БОТ ПО ВКР

Назначение Telegram-бота:
Telegram-бот создан как AI-помощник по ВКР/БКП. Он должен отвечать на вопросы о дипломной работе, теме проекта, архитектуре, модулях, технологиях, базе данных, интерфейсе, тестировании, эффективности и пользовательском сценарии.

Технологии Telegram-бота:
1. Python — язык программирования бота.
2. python-telegram-bot — библиотека для подключения Telegram Bot API.
3. Ollama — локальная платформа для запуска языковой модели.
4. Локальная языковая модель — используется для генерации ответов без платного OpenAI API.
5. knowledge_base.txt — текстовая база знаний проекта.
6. VS Code — среда разработки.
7. .env — файл для хранения Telegram token и настроек модели.

Почему используется Ollama:
Ollama используется потому, что позволяет запускать языковую модель локально на компьютере без оплаты OpenAI API. Это удобно для учебного проекта, так как бот может работать бесплатно, использовать локальную базу знаний и отвечать на вопросы о ВКР.

Как работает Telegram-бот:
1. Пользователь отправляет вопрос в Telegram.
2. Python-бот получает сообщение через Telegram Bot API.
3. Бот читает файл knowledge_base.txt.
4. Вопрос пользователя и база знаний передаются локальной модели Ollama.
5. Модель формирует ответ на основе базы знаний.
6. Бот отправляет ответ пользователю в Telegram.

Роль Telegram-бота в проекте:
Telegram-бот является дополнительным интеллектуальным помощником по ВКР. Он не заменяет основное приложение VSTU READS, а помогает быстро отвечать на вопросы о проекте, объяснять архитектуру, функции, технологии, тестирование и результаты работы.

Чем Telegram-бот отличается от VSTU READS:
VSTU READS — это основное настольное приложение для анализа русскоязычных текстов и выделения тематических групп.
Telegram-бот — это справочный AI-помощник, который отвечает на вопросы о ВКР и помогает объяснять проект.

Важно:
Если пользователь спрашивает о фактах, которых нет в базе знаний, бот должен честно сообщить, что информации недостаточно и нужно добавить данные в knowledge_base.txt. Бот не должен придумывать неподтвержденные детали.
